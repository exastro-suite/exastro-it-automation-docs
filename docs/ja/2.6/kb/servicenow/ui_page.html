ServiceNow UI Pageを用いたIT Automation画面の実装手順¶

はじめに¶

本書では、ServiceNow UI Pageを用いてIT Automation画面を構築し、IT Automation APIとの連携を含めた実装方法について説明します。
主にServiceNow PDI（個人開発者インスタンス）を前提に解説しますが、他の環境でも適用可能な内容としています。

条件¶

バージョン¶

  • IT Automationのバージョン: 2.6.2
  • ServiceNowインスタンスのバージョン: Zurich

接続要件および事前確認事項¶

  • ServiceNowインスタンスからIT AutomationサーバへのHTTPS通信が許可されていること
  • IT AutomationのAPIエンドポイントに到達可能であること
  • 必要に応じてプロキシ設定が正しく構成されていること
  • IT Automation側で認証（Basic認証）が有効であること

概要図¶

実装手順¶

①資材ダウンロード¶

下記より資材をダウンロードしてください。
https://github.com/exastro-suite/exastro-it-automation/raw/refs/heads/2.6/knowledge_base/servicenow/ita_snow_ui_page_materials.zip

②基本認証構成登録¶

  1. ヘッダーメニューのAllから「System Web Services / Outbound / REST message」にアクセス
  2. NEWボタンから新規作成

    Name

    IT Automation REST Message

    Accessible from

    This application scope only

    Endpoint

    https://
  3. Authenticationの設定

    Authentication type

    Basic
  4. Basic auth profileを追加（サーチアイコンを押下）
  5. 新しいWindowが開くのでNEWボタンから新規作成

    Name

    IT Automation Basic Auth

    Username

    {{ IT AutomatinのログインID }}

    Password

    {{ IT Automationのログインパスワード }}
  6. SubmitボタンでBasic Authを登録
  7. REST Message画面に戻り、SubmitボタンでREST Messageを登録
  8. 一覧に戻るので、登録したIT Automation REST Messageを開く
  9. インフォアイコンからOpen Recordボタンを押下
  10. 登録したBasic Authのレコードが開くので、左上のメニューからCopy sys_idで参照に必要なIDを控えておく

IT AutomationのAPIエンドポイントに到達可能かテスト¶

※不要であればこの項目はスキップしてください。
  1. 登録したIT Automation REST Messageを開く
  2. HTTP Methodsに登録してあるDefault GETを開く
  3. Endpointにユーザの情報を取得するエンドポイントを設定

    Endpoint

    {{ IT Automation URL }}/api/{{ オーガナイゼーションID }}/workspaces/{{ ワークスペースID }}/ita/user/
  4. Updateボタンで更新
  5. ページが戻るので、再度Default GETを開く
  6. Updateボタンの下にあるRelated LinksのTestを押下
  7. 正常ならHTTP statusが200でResponseに結果が返却される

③スクリプト済み REST API設定¶

  1. ヘッダーメニューのAllから「System Web Services / Scripted Web Services / Scripted REST APIs」にアクセス
  2. NEWボタンから新規作成

    Name

    IT Automation API

    API ID

    it_automation
  3. Securityの設定

    Default ACLs

    Scripted REST External Default
  4. Submitボタンで登録
  5. 一覧に戻るので、登録したIT Automation APIを開く
  6. 後の項目で必要なBase API pathを控えておく
  7. ResourcesのNEWボタンからリソースを新規作成

    Name

    dispatch

    HTTP method

    POST

    Relative path

    /dispatch

    Script

    {{ ①でダウンロードした資材から「scripted_rest_api.js」の内容を張り付ける }}
  8. Submitボタンでリソースを登録

注釈

scripted_rest_api.js内で、「④テーブル設定」で登録するテーブル名（Name）・列名（Column name）を指定しています。本書の指定とは別のものにした場合は下記の部分を修正してください。
3～6行目:
var configTableName = 'u_it_automation_config'; ※テーブル名（IT Automation Config）
var tempTableName = 'u_it_automation_temp_file'; ※テーブル名（IT Automation Temp File）
var urlColumnName = 'u_it_automation_url'; ※列名（IT Automation URL）
var basicAuthColumnName = 'u_basic_auth_sys_id'; ※列名（Basic Auth Sys ID）

  3. ColumnsのInsert a new row...から必要な項目を追加（※Max lengthは適宜調整してください）

    Column label

    Type

    Max length

    User Name

    String

    64

    IT Automation URL

    String

    256

    String

    256

    Organization ID

    String

    64

    Workspace ID

    String

    64

    Basic Auth Sys ID

    String

    Base API path

    String

    32
  4. Submitボタンでテーブルを登録
  5. ヘッダーメニューのAllから登録した「IT Automation Config / IT Automation Configs」にアクセス
  6. NEWボタンからIT Automation接続情報を新規作成

    Basic Auth Sys ID

    Web Assets URL

  7. SubmitボタンでIT Automation接続情報を登録

IT Automationとのファイルのやり取りで使用するテーブルの作成¶

IT Automationのファイルを表示したりダウンロードした際にファイルを一時的に保存するテーブルです。
  1. ヘッダーメニューのAllから「System Definition / Tables」にアクセス
  2. NEWボタンからテーブルを新規作成

    Label

    IT Automation Temp File

    Name

    u_it_automation_temp_file
  3. Submitボタンでテーブルを登録（列の追加は必要ありません）
  4. ヘッダーメニューのAllから「System Definition / Table Rotations」にアクセス
  5. NEWボタンからテーブルローテーションを新規作成（※クリアする期間などは適宜調整してください）

    Name

    u_it_automation_temp_file

    Type

    Rotation

    Rotations

    1

    Duration

    Days 1
  6. Submitボタンでテーブルローテーションを登録

⑤UIページ設定¶

  1. ヘッダーメニューのAllから「System UI/ UI Pages」にアクセス
  2. NEWボタンからUIページを新規作成

    Name

    IT Automation UI Page

    HTML

    {{ ①でダウンロードした資材から「ui_page.html」の内容を張り付ける }}

    Client script

    {{ ①でダウンロードした資材から「ui_page.js」の内容を張り付ける }}
  3. クリップアイコンを押下
  4. ①でダウンロードした資材から下記をAttachmentsに追加する

    • snow_patch.js

    • snow_patch.css

    • snow_create_menu.js

    • snow_dashboard_widget.js

    • snow_table_worker.js

    • messageid_ja.js

  5. Attachmentsに追加したファイル名を右クリック「リンクのアドレスをコピー」でURLをコピーし下記の/sys_attachment.do?sys_id=********の部分を対応したファイルと入れ替える
    HTML 9～12行目:

    createMenu: '/sys_attachment.do?sys_id=********',

    snow_create_menu.js

    tableWorker: '/sys_attachment.do?sys_id=********',

    snow_table_worker.js

    widget: '/sys_attachment.do?sys_id=********',

    snow_dashboard_widget.js

    ja: '/sys_attachment.do?sys_id=********'

    messageid_ja.js
    HTML 17～18行目:

    var patch = '/sys_attachment.do?sys_id=********';

    snow_patch.js

    var cssSysAtt = '/sys_attachment.do?sys_id=********';

    snow_patch.css
  6. Submitボタンを押下
  7. Roleの選択が出てくるので「admin」を選択
  8. {{ Instance URL }}ui_page.do?sys_id={{ 作成したUI PageのSys ID }}でページを開く

注釈

ui_page.html内で、「④テーブル設定」で登録するテーブル名（Name）・列名（Column name）を指定しています。本書の指定とは別のものにした場合は下記の部分を修正してください。
21行目:
var gr = new GlideRecord('u_it_automation_config'); ※テーブル名（IT Automation Config）
31～35行目:
itaUrl = gr.getValue('u_it_automation_url'); ※列名（IT Automation URL）
webAssetsUrl = gr.getValue('u_web_assets_url'); ※列名（Web Assets URL）
orgId = gr.getValue('u_organization_id'); ※列名（Organization ID）
wsId = gr.getValue('u_workspace_id'); ※列名（Workspace ID）
apiPath = gr.getValue('u_base_api_path'); ※列名（Base API path）

注釈

ヘッダーの色をServiceNowカラーに変更しています。必要なければUI Page HTMLの92～98行目を削除してください。

注釈

Try Itからも画面表示可能ですが、iframeでの表示になりその場合はIT Automationのヘッダーとサイドメニューが表示されません。

制限事項¶

  • ServiceNowのREST Message（sn_ws.RESTMessageV2）では、リクエストおよびレスポンスで扱えるデータサイズに制限があり、最大約10MBまでとなります。
    この制限を超えるデータを送受信した場合、エラーの発生などにより正常に処理できない可能性があります。
    また、ファイルの送信においては、multipart/form-dataには対応していないため、バイナリデータはBase64形式に変換して送信されます。
    その際、データサイズは約1.3倍となるため、実際に扱えるファイルサイズはこれを考慮する必要があります。
